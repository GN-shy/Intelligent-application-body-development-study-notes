

# HTML 5笔记
## 一、HTML 介绍
### 1. 核心定义与特性
HTML 全称为“超文本标记语言”，是一种由预定义标签构成的语言，核心特点是“超越文本限制”——除了承载文字信息，还能嵌入图片、音频、视频等多媒体内容。
HTML 无需编译，编写完成后直接在浏览器上运行，标签由浏览器自动解析，无需额外编译工具或步骤。
HTML 文件可在任意位置创建，文件后缀名固定为 `.html` 或 `.htm`，两种后缀功能完全一致，仅为命名习惯差异。

### 2. W3C 标准(网页组成)
根据 W3C 标准，网页主要由三部分协同构成，三者分工明确，共同实现网页的结构、样式与交互：
- **结构层**：由 HTML 实现，相当于网页的“骨骼”，负责定义页面内容的组织结构(如标题、段落、表格、图片位置等)，决定“页面有什么”。
- **表现层**：由 CSS 实现，相当于网页的“化妆师”，负责控制页面内容的视觉样式(如字体颜色、大小、布局间距、背景色等)，决定“页面看起来怎么样”。
- **行为层**：由 JavaScript 实现，负责实现页面的交互逻辑(如点击按钮跳转、表单验证、动态加载内容等)，决定“页面能做什么”。

### 3. 基础语法规则
- 标签大小写：HTML 标签区分大小写，例如 `<HTML>` 与 `<html>`、`<Body>` 与 `<body>` 视为不同标签，开发中需注意标签大小写的一致性。
- 属性值引号：HTML 标签的属性值使用单引号或双引号包裹均可，两种写法均有效(如 `color='red'` 与 `color="red"` 功能一致)。

## 二、HTML 基本格式
HTML 文档有固定的结构框架，核心由“根标签”“头标签”“体标签”组成，具体基础结构如下：
```html
<!-- 根标签:整个 HTML 文档的最外层容器,所有内容必须包裹在 <html> 与 </html> 之间 -->
<html>
<!-- 头标签:用于定义文档的元信息(如文档标题、引入外部资源等),内容不直接显示在页面中 -->
<head>
<!-- 标题标签:定义文档的标题,显示在浏览器窗口的标签栏上,每个 HTML 文档仅一个 <title> 标签 -->
<title>文档标题</title>
</head>
<!-- 体标签:定义文档的主体内容,所有需要在浏览器页面中直接显示的内容(文字、图片、按钮等)均需放在此处 -->
<body>
页面主体内容(会直接显示在浏览器中)
</body>
</html>
```

### HTML5 标准基础代码示例
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width,initial-scale=1.0">
    <title>Document</title>
</head>
<body>

</body>
</html>
```

### 核心元素说明
1. **文档类型声明**：`<!DOCTYPE html>` 是HTML5的文档类型声明，告诉浏览器当前页面是使用HTML5规范编写的。
2. **html元素**：`<html>` 为根元素，包裹页面中所有的内容；`lang` 属性用于声明网页的主要语言，帮助浏览器、搜索引擎等正确处理页面内容（`en`代表英语，`zh-CN`代表中文），示例：`<html lang="zh-CN" class="gray">`。
3. **head元素**：头部元素，包含了文档的元(meta)数据，主要保存供机器处理的信息，而非人类可读信息。
4. **字符集设置**：`<meta charset="UTF-8">` 将文档的字符集设置为UTF-8，UTF-8包含绝大多数人类书面语言的大多数字符，不加此设置可能会引起页面乱码。
5. **移动端页面适配**：`<meta name="viewport" content="width=device-width,initial-scale=1.0">` 能确保网页在移动设备上以最佳状态呈现，提升用户浏览体验。
6. **title元素**：设置页面的标题，显示在浏览器的标签栏中，示例：`<title>哔哩哔哩(*-*)干杯-bilibili</title>`。

## 四、标题和段落
### 1. 标题标签 `<h>`
- 仅有六个层级，`h1` 到 `h6`。
- 显示特点：标题文字会加粗显示，并且每行只显示一个。
- 使用规范：最好只对每个页面使用一次`h1`（顶级标题，如新闻的标题、网页的logo部分）；六个标题层次中，每页最好不超过使用三个。

### 2. 段落标签 `<p>`
- HTML 元素表示文本的一个段落，语法：`<p>里面是段落文本</p>`。
- 显示特点：每行只显示一个，文字显示不开会自动换行。

### 3. 换行标签 `<br>`
实现文本的强制换行，为单标签。

### 语义化说明
根据内容的结构和含义选择恰当的HTML元素，好处：
1. 更清晰的代码结构
2. 对搜索引擎更友好
3. 更好的可访问性

## 八、图像标签
### 基础语法
图像标签为单标签（空元素），核心语法：`<img src="" alt="">`，示例：`<img src="pig.jpg" alt="这是佩奇">`。

### 核心属性
默认包含两个核心属性，属性采取键值对形式`属性="值"`，属性之间无先后顺序，但必须用空格分隔：
| 属性 | 作用                                                       |
| ---- | ---------------------------------------------------------- |
| src  | 指向要插入到页面的图像地址                                 |
| alt  | 备选文本，用于在图片无法显示或者因为网速慢情况下显示的文字 |

### 其他属性
| 属性   | 作用                         |
| ------ | ---------------------------- |
| width  | 设置图片宽度(建议css修改)    |
| height | 设置图片高度(建议css修改)    |
| title  | 图像标题，鼠标悬停显示的文本 |

### 常见的图片格式及适用场景
| 格式     | 特性                                                  | 适用场景                                        |
| -------- | ----------------------------------------------------- | ----------------------------------------------- |
| jpeg/jpg | 有损压缩技术，放大缩小会变得模糊或有锯齿              | 摄影照片、网页图片 (非透明背景)                 |
| png      | 无损压缩，支持透明度                                  | Logo、网页图形、需要透明度的图像                |
| gif      | 支持动画，最多256色(索引色)，支持简单动画和透明背景   | 简单动画、表情包、低色彩图形                    |
| webp     | Google开发的现代格式，支持有损/无损压缩、透明度和动画 | 网页优化(替代JPEG/PNG/GIF)，如淘宝等平台        |
| avif     | 基于AV1视频编码，支持高压缩率和HDR，压缩效率优于WebP  | 未来网页优化，需高性能压缩的场景，如B站、京东等 |

### 图片格式选择总结
- 网页优化：优先WebP/AVIF(兼容时)，备选JPEG/PNG。
- 透明图像：PNG(静态)或WebP(动态)。
- 动画：WebP/GIF(简单动画)。
- 开发说明：图片还有其他格式（如svg、HEIC等），开发中设计师提供图片，直接使用即可。

### 路径
在HTML中，路径用于指定文件(如图像、样式表或其他网页)的位置，即图片相对于html文件的位置，分为**相对路径**和**绝对路径**。
#### 1. 相对路径
相对于当前文件位置的路径，是开发中常用的路径方式：
- 同一目录：直接使用文件名即可，`./` 表示当前文件夹，示例：`<img src="pig.jpg" alt="这是佩奇">`、`<img src="./pig.jpg" alt="这是佩奇">`。
- 下级子目录(重点)：使用`目录名/文件名`，示例：`<img src="img/pig.jpg" alt="这是佩奇">`。
- 上级目录：使用`../`返回上一级，示例：`<img src="../pig.jpg" alt="这是佩奇">`。

#### 2. 绝对路径
从根目录开始的完整路径，包含完整的URL地址（了解即可）：
- 从盘符开始：`<img src="E:\HTML5\代码\pic.jpg" alt="示例图片">`。
- 完整网络地址：`<img src="https://www.example.com/images/pic.jpg" alt="示例图片">`。

## 九、视频和音频标签
### 1. 视频标签 `<video>`
#### 基础语法
`<video src=""></video>`，示例：`<video src="video.mp4" controls width="300"></video>`。

#### 核心属性
| 属性         | 作用                       |
| ------------ | -------------------------- |
| src          | 指向要插入到页面的视频地址 |
| controls     | 显示浏览器自带播放控件     |
| width/height | 设置视频的宽度和高度       |
| autoplay     | 自动播放                   |
| loop         | 循环播放                   |
| muted        | 静音                       |
| poster       | 预览图像                   |

#### 格式建议
优先考虑MP4格式，兼容性最佳。

#### 兼容性写法
```html
<video controls>
    <source src="video.mp4" type="video/mp4">
    <source src="video.ogg" type="video/ogg">
    <source src="video.webm" type="video/webm">
    <p>您的浏览器不支持HTML5 Video标签。请升级浏览器。</p>
</video>
```
**说明**：
1. 将src属性放在几个单独的`<source>`元素当中，这些元素分别指向各自的资源。
2. 浏览器会检查`<source>`元素，并且播放第一个与其自身相匹配的媒体。
3. 每个`<source>`元素都含有type属性，浏览器会通过检查这个属性来迅速跳过那些不支持的格式；若未添加type属性，浏览器会尝试加载每一个文件，直到找到能正确播放的格式，会消耗大量的时间和资源。

#### 开发写法
```html
<video autoplay poster="">
    <source src="video.mp4" type="video/mp4">
    <p>您的浏览器不支持HTML5 Video标签，请升级浏览器。</p>
</video>
```

### 2. 音频标签 ``
#### 基础说明
一般优先使用MP3格式，核心属性与视频标签一致（`autoplay`/`loop`/`muted`），无`width`/`height`/`poster`属性。
#### 基础语法
`<audio src="">`，支持`<source>`做兼容性处理，写法与视频标签一致。

## 十、超链接标签 `<a>`
超链接能够将文档链接到任何其他文档(或其他资源)，也可以链接到文档的指定部分。

### 基础语法
`<a href=""></a>`，示例：`<a href="https://www.deepseek.com/">DeepSeek官网</a>`。

### 核心属性
| 属性   | 作用            | 说明                                              |
| ------ | --------------- | ------------------------------------------------- |
| href   | 超文本引用/目标 | 包含跳转的网址/路径，为核心属性                   |
| title  | 提示文字        | 鼠标悬停时显示的文本                              |
| target | 打开页面的方式  | `_self`：当前窗口打开(默认)；`_blank`：新窗口打开 |

### 不同类型的超链接
1. **下载链接**：若`href`指向exe、压缩包等文件，点击后会触发下载，示例：`<a href="downLoad.exe">下载window版本</a>`。
2. **空链接**：没有实际指向目标的超链接，用`#`表示，示例：`<a href="#">商品列表</a>`。
3. **邮件链接**：需以`mailto:`开头，示例：`<a href="mailto:pigmozilla.org">给我发件</a>`；**注意**：仅适用于简单场景/个人使用，公司/流量大的网站慎用，存在垃圾邮件风险，建议用其他方案替代。
4. **锚点链接**：允许用户在同一个页面内跳转到指定位置，适合长页面导航，实现步骤：
   - 定义锚点目标：使用`id`属性，示例：`<h2 id="1">第一部分</h2>`。
   - 创建跳转链接：使用`#`标记锚点目标，示例：`<a href="#1">跳转到第一部分</a>`。
   - 页面滑动效果：在`<head>`标签中添加CSS代码，实现点击跳转的平滑效果：
     ```html
     <style>
         html{
             scroll-behavior: smooth;
         }
     </style>
     ```

## 十一、布局标签
网页外观多样，但核心结构通常包含：页眉、导航栏、主内容、侧边栏、页脚等，布局标签分为**语义化布局标签**、**无语义标签**和**列表标签**。

### 1. 语义化网站结构标签
| 标签        | 作用                             |
| ----------- | -------------------------------- |
| `<header>`  | 网页页眉(头部)                   |
| `<main>`    | 网页核心内容，每个页面只能有一个 |
| `<nav>`     | 导航栏                           |
| `<article>` | 文章相关内容容器                 |
| `<section>` | 页面内容分块容器                 |
| `<aside>`   | 侧边栏                           |
| `<footer>`  | 页面页脚(底部)                   |

### 2. 无语义标签
无合适语义标签时，用于布局的兜底标签，无默认语义：
#### (1) div标签
- 块级元素：默认独占一行，前后会自动换行。
- 通常用于布局结构，作为其他元素的容器。
- 可以包含其他块级或行内元素。

#### (2) span标签
- 行内元素：不会换行，仅包裹内容的一部分。
- 用于对文本或行内元素的局部样式或操作。

### 3. 列表标签
HTML列表是网页内容组织的重要元素，能让内容显示更加整齐有序，分为**无序列表**、**有序列表**、**描述列表**三类。
#### (1) 无序列表 `<ul>`
- 适用于顺序无关紧要的列表，默认带项目符号。
- 语法：`<ul>` 为列表容器，仅能包含`<li>`元素；`<li>` 为列表选项，可嵌套其他HTML元素。
  ```html
  <ul>
      <li>猪爸爸</li>
      <li>猪妈妈</li>
      <li>佩奇</li>
      <li>乔治</li>
  </ul>
  ```
- 适用场景：常用于整齐对齐的模块展示。

#### (2) 有序列表 `<ol>`
- 适用于顺序有关紧要的列表，默认带数字序号。
- 语法：`<ol>` 为列表容器，仅能包含`<li>`元素；`<li>` 为列表选项，可嵌套其他HTML元素。
  ```html
  <ol>
      <li>看视频</li>
      <li>写代码</li>
      <li>做笔记</li>
      <li>多复习</li>
  </ol>
  ```

#### (3) 描述列表 `<dl>`
- 用于标记一组项目及相关描述，由`<dl>`/`<dt>`/`<dd>`组成。
- 语法：
  ```html
  <dl>
      <dt>家电</dt>
      <dd>电视</dd>
      <dd>冰箱</dd>
      <dd>空调</dd>
      <dd>烟灶</dd>
  </dl>
  ```
- 标签说明：
  - `<dl>`：定义列表的容器，仅能包含`<dt>`和`<dd>`元素。
  - `<dt>`：定义被描述的术语，通常显示为左对齐或加粗，一个`<dt>`可对应多个`<dd>`。
  - `<dd>`：包含术语的定义或描述，通常显示为缩进形式，可包含段落、图片、链接等其他HTML元素。

## 十二、表格标签
### 核心作用
以结构化方式展示行列数据，使信息清晰、易读且便于对比；**适用场景**：主要用于数据展示、后台管理系统的信息展示。

### 1. 表格基本组成标签
| 标签      | 作用                                         |
| --------- | -------------------------------------------- |
| `<table>` | 表格容器标签，包裹所有表格内容               |
| `<tr>`    | 行标签，定义表格的一行                       |
| `<td>`    | 单元格标签，定义表格的普通单元格             |
| `<th>`    | 表头单元格，定义表格的表头，文字默认加粗居中 |

### 2. 表格结构标签
增强表格语义，让表格结构更加清晰，按需使用：
| 标签      | 作用                   |
| --------- | ---------------------- |
| `<thead>` | 定义表格的头部区域     |
| `<tbody>` | 定义表格的主体内容区域 |
| `<tfoot>` | 定义表格的底部区域     |

**示例**：
| 姓名   | 成绩 | 对应结构标签 |
| ------ | ---- | ------------ |
| 小美   | 100  | `<thead>`    |
| 大壮   | 98   | `<tbody>`    |
| 奥德彪 | 80   | `<tbody>`    |
| 平均分 | 92.6 | `<tfoot>`    |

### 3. 合并单元格
**开发说明**：表格开发中很少使用合并，会导致表格难以维护，且可能影响响应式适配(尤其在移动端)，特殊情况需使用时可借助AI。
#### 合并原理
1. 确定是**跨行合并(rowspan)** 还是**跨列合并(colspan)**。
2. 找到目标单元格（左上原则），书写合并数量`rowspan="n"`/`colspan="n"`。
3. 删除合并后产生的多余单元格。
- 跨行合并：`rowspan="2"`（纵向合并2个单元格）
- 跨列合并：`colspan="2"`（横向合并2个单元格）

## 十三、表单
### 核心作用
收集用户输入数据，并将数据提交到后端进行处理。
**常见场景**：用户登录/注册、搜索框、联系表单、问卷调查、订单支付、文件上传等。

### 表单核心组成三部分
1. **表单容器**：`<form>`，包裹所有表单控件。
2. **表单控件**：包含`<input>`通用输入控件、`<textarea>`多行文本输入框、`<select>`下拉选择框、`<button>`自定义按钮等。
3. **辅助标签**：`<label>`，关联输入控件的文本标签，提升可访问性。

### 1. 表单容器 `<form>`
- 作用：定义表单的容器，包裹所有表单控件，语法：`<form action=""></form>`。
- `action`属性：定义提交表单时，将收集的数据发送至哪个URL进行处理；未涉及数据交互时，属性值可留空。

### 2. 表单控件
#### (1) 通用输入控件 `<input>`
是最常用的表单元素，通过`type`属性定义输入框类型，核心`type`属性值：
| type属性值 | 说明                   |
| ---------- | ---------------------- |
| text       | 单行文本框             |
| password   | 密码框（输入内容隐藏） |
| radio      | 单选框                 |
| checkbox   | 复选框                 |
| file       | 文件域（文件上传）     |

**通用属性**：
| 其他属性     | 说明                                   |
| ------------ | -------------------------------------- |
| placeholder  | 输入框提示信息，输入内容后消失         |
| name         | 元素的名称，用于后端接收数据           |
| maxlength    | 允许输入的最大字符数                   |
| accesskey    | 使元素获得焦点的快捷键                 |
| autocomplete | 控制表单的自动填充行为，取值`on`/`off` |

**单选框/复选框专属属性**：
| 其他属性 | 说明                                              |
| -------- | ------------------------------------------------- |
| name     | 实现分组，同组的单选框/复选框需设置相同的`name`值 |
| value    | 表单提交的实际值                                  |
| checked  | 设置控件默认选中，无需赋值                        |

**文件域`file`专属属性**：
| 其他属性 | 说明                                       |
| -------- | ------------------------------------------ |
| multiple | 允许选择多个文件，无需赋值                 |
| accept   | 规定选择的文件类型，多个类型中间用逗号分隔 |

#### (2) 多行文本输入框 `<textarea>`
也称为文本域，适用于用户输入大量自由格式文本的场景（如评论、反馈）。
- 基础语法：`<textarea></textarea>`
- 常见属性：
| 常见属性    | 说明                          |
| ----------- | ----------------------------- |
| name        | 表单名称，用于后端接收数据    |
| placeholder | 输入框提示信息                |
| rows        | 文本显示行数，正整数，默认为2 |
| cols        | 文本显示列数，正整数，默认20  |

#### (3) 下拉选择框 `<select>`+`<option>`
- `<select>`：下拉框容器，语法：`<select name="" id="">`。
- `<option>`：下拉框的每个选项，需设置`value`属性（提交的实际值），语法：`<option value="北京">北京</option>`。
- 默认选中：给`<option>`添加`selected`属性，无需赋值。
- 示例：
  ```html
  <select name="" id="">
      <option value="北京">北京</option>
      <option value="深圳">深圳</option>
      <option value="上海">上海</option>
      <option value="广州">广州</option>
  </select>
  ```

#### (4) 自定义按钮 `<button>`
- 作用：定义表单按钮，元素内部可放置文本、图像等内容。
- 基础语法：`<button>搜索</button>`。
- 常用属性：`disabled`，禁用按钮，添加后按钮无法点击，无需赋值。

### 3. 辅助标签 `<label>`
- 作用：表示用户界面中某个元素的说明，**提升可访问性**（点击标签可聚焦对应输入框），提升表单用户体验。
- 两种使用方式：
  1. 利用`for`和`id`相关联：`<input type="radio" id="nan" name='sex'>` + `<label for='nan'>男</label>`。
  2. 直接包裹控件：`<label>男 <input type= radio name='sex'></label>`。

## 十四、字符实体
字符实体是一段以连字号(`&`)开头、以分号(`;`)结尾的文本(字符串)，常用于显示HTML保留字符和不可见的字符(如“不换行空格”)。

| 字符 | 实体       | 说明                                        |
| ---- | ---------- | ------------------------------------------- |
| &    | `&amp;`    | 解析为实体或字符引用的开头、解析为tag的开头 |
| >    | `&gt;`     | 解析为tag的结尾                             |
| "    | `&quot;`   | 解析为attribute的值的开头和结尾             |
|      | `&nbsp;`   | 解析为不换行空格                            |
| –    | `&ndash;`  | 解析为短破折号(等于em单位的一半宽度)        |
| —    | `&emdash;` | 解析为长破折号(等于"m"字符的宽度)           |
| ©    | `&copy;`   | 解析为版权符号                              |
| ®    | `&reg;`    | 解析为注册商标符号                          |
| ™    | `&trade;`  | 解析为商标符号                              |
| ≈    | `&asymp;`  | 解析为近似等于符号                          |
| ≠    | `&ne;`     | 解析为不等于符号                            |
| £    | `&pound;`  | 解析为英镑符号                              |
| €    | `&euro;`   | 解析为欧元符号                              |
| °    | `&deg;`    | 解析为度符号                                |
