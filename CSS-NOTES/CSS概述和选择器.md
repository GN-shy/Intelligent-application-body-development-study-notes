#                                                      CSS概述和选择器

## 一 概述

CSS（层叠样式表）。是用来控制网页在浏览器中的显示外观的语言。 

属性：采取键值对形式。属性名： 属性值；

书写规范：

- 选择器和大括号中间保留一个空格的距离
- 属性名和属性值中间也要保留一个空格
- 每个css属性单独占一行。

## 二、CSS分类

### 内联样式表(行内样式表)

样式写到标签内部，可以控制当前标签的样式，特殊情况使用。

```
< p style="color: red;font-size: 14px;">文字内容< /p > 
```

### 内部样式表

写到< head >标签中，脱离结构，可以控制当前页面的所有标签，较为常用

### 外部样式表

单独新建一个css文件，完全脱离结构，可以控制整个网站的所有标签，最常用

```
< link rel="stylesheet" href="./css/index.css" >
```

## 三、CSS选择器

CSS选择器是CSS规划的第一部分。它是选择HTML元素的方式。

### 基础选择器

基础选择器指的是有单个选择器组成，常见的有：

   #### 类型选择器

类型选择器选择某个类型，也称为标签选择器

**定义**：直接匹配 HTML 文档中的**标签名**，作用于页面中所有该类型的元素。

**语法**：直接写标签名（如 `div`、`p`、`h1`），无需任何前缀。

```
p {
    color: pink;
}

```

CSS层叠性，CSS规则可以同时作用于一个元素，浏览器通过特定规则决定最终生效的样式。层叠性解决了样式冲突的问题。（原则：采用就近原则，后定义的样式覆盖先前的样式）

#### 类选择器

类选择器选择某一个元素或多个元素

```
.box {
    color: pink;
}
```

语法：

```
.类名 {
    属性: 值;
}
```

```
<标签 class="类名">内容</标签>
```

（HTML中）

注意事项：

- 类名自定义，但不能是中文，纯数字
- 多个英文单词组成尽量用-连接
- 命名要有意义，尽量见名知意
- class属性可以有多类名中用空格隔开

#### id选择器

选择HTML中具有特定id属性的唯一元素

```
#box {
    color: pink;
}
```

语法:

```
#id名{
    样式声明；
}
```

```
< 标签 id="" >~~</标签>
```

作用：选择特定id属性的唯一元素

特点：同一页面中，id值必须唯一

场景：后期主要是配合Javascript添加交互效果

#### 通配符选择器

通配符选择器可以选择HTML中所有的标签，进行样式修改

```
* {
   Style;
}
```

特殊情况下通过通配符清除所有元素的默认边距和填充，统一不同浏览器的默认样式



#### 总结



| 选择器类型   | 语法示例          | 匹配范围                    | 复用性       | 典型使用场景                             | 注意事项                       |
| ------------ | ----------------- | --------------------------- | ------------ | ---------------------------------------- | ------------------------------ |
| 类型选择器   | `div { ... }`     | 匹配所有指定 HTML 标签元素  | 某一类型标签 | 全局样式重置、基础标签样式（如 body, p） | 避免滥用，可能导致样式冲突     |
| 类选择器     | `.nav { ... }`    | 匹配所有含指定 class 的元素 | 多次使用     | 多元素共享样式                           | 优先使用，避免与 ID 选择器冲突 |
| ID 选择器    | `#header { ... }` | 匹配唯一含指定 id 的元素    | 唯一性       | 唯一元素标识                             | 必须唯一，避免样式覆盖风险     |
| 通配符选择器 | `* { ... }`       | 匹配所有元素                | 全局覆盖     | 全局样式重置                             | 性能较差，慎用                 |

> 💡 提示：大部分情况下，**类选择器** 是最推荐、最重要的选择器类型。





### 关系选择器

关系选择器是通过**位置关系**来选择目标元素（标签）。常见的有：

#### 后代选择器

选择某个元素的后代元素（不限层级，包括子元素，孙元素）

HTML

```html
<div>
    <p>我是文字</p>
</div>
<p>我是文字</p>
```

CSS 样式

```css
div p {
    color: pink;
}
```

- **语法**：`父级选择器 子级选择器`，中间用**空格**隔开。
- **作用**：仅选择**父级元素内部**的子元素（包括嵌套更深的后代元素）。
- **效果**：只有 `<div>` 内部的 `<p>` 文字会变成粉色，外部的 `<p>` 不受影响。
- **灵活性**：父级和子级都可以是**任意类型的选择器**（类选择器、ID选择器等）

#### 子代选择器

选择某个元素的直接子元素（仅限一层）

语法：

```css
父元素 > 子元素 {
  /* 样式规则 */
}
```



**HTML 结构：**
```html
<div>
  <span>我是儿子</span>
  <p>
    <span>我是孙子</span>
  </p>
</div>
```

**CSS 样式：**
```
div > span {
  color: pink;

```

- 只会选中 `div` 直接包裹的**第一层** `<span>`（即“亲儿子”元素），文本变为粉色。
- 不会选中嵌套在 `<p>` 里的 `<span>`（即“孙子”元素），因为它不是 `div` 的直接子元素。


- **子代选择器 (`>`)**：只匹配**直接子元素**（亲儿子），不匹配更深层的后代。
- **后代选择器（空格）**：会匹配所有层级的后代元素（包括儿子、孙子、曾孙等）。

#### 兄弟选择器

兄弟选择器分为邻接兄弟选择器和通用兄弟选择器

##### 邻接兄弟选择器 (`+`)

- **语法**：`元素A + 元素B`
- **作用**：选中**紧跟在元素A后面**的**第一个**元素B（必须是同一个父级下的直接兄弟）。

**示例代码：**
```html
<div class="box">
  <p>文字内容</p>
  <p>文字内容</p>
  <p>文字内容</p>
  <h2>标题</h2>
  <p>文字内容</p> <!-- 会被选中 -->
  <p>文字内容</p>
  <p>文字内容</p>
</div>
```

```css
h2 + p {
  color: pink;
}
```
- **效果**：只有 `<h2>` 后面**第一个** `<p>` 文字会变成粉色。

   ##### 通用兄弟选择器 (`~`)


- **语法**：`元素A ~ 元素B`
- **作用**：选中**跟在元素A后面**的**所有**元素B（同一个父级下，无需紧挨着）。

**示例代码：**
```css
h2 ~ p {
  color: pink;
}
```
- **效果**：`<h2>` 后面的**所有** `<p>` 文字都会变成粉色（共3个 `<p>` 元素）。

核心区别对比

| 选择器   | 语法    | 匹配范围         | 示例效果           |
| -------- | ------- | ---------------- | ------------------ |
| 邻接兄弟 | `A + B` | 仅A后**第一个**B | 只选中h2后的第1个p |
| 通用兄弟 | `A ~ B` | A后**所有**B     | 选中h2后的所有p    |



#### 关系选择器总结

| 选择器         | 语法    | 选择范围             | 实例                              |
| -------------- | ------- | -------------------- | --------------------------------- |
| 后代选择器     | `A B`   | 所有后代（跨层级）   | `ul li {color: pink;}`            |
| 子代选择器     | `A > B` | 直接子元素（仅一层） | `div>span {color: pink;}`         |
| 邻接兄弟选择器 | `A + B` | 紧邻的下一个同级元素 | `h2+p {}` 选中标题后的第一个p元素 |
| 通用兄弟选择器 | `A ~ B` | A之后的所有同级元素  | `h2~p {}` 选中标题后的所有p元素   |

- **后代选择器**：是日常开发中最常用的选择器，能匹配父元素下所有层级的子元素，无论嵌套多深。
- **子代选择器**：仅匹配父元素的**直接子元素**，不会作用于更深层的嵌套元素，适合精准控制结构。
- **邻接兄弟选择器**：只作用于目标元素后**紧挨着的第一个**同级元素，范围最窄。
- **通用兄弟选择器**：作用于目标元素后**所有符合条件**的同级元素，范围更广。




### 分组选择器（并集选择器）

是将不同的选择器组合在一起，使用逗号分割。

使用场景：

- 多个元素具备相同样式

语法

```
div,
span{
    color: pink;
}
```



### ***伪类选择器***

选择元素的特定状态或结构位置，符号是冒号：

#### 状态伪类

##### 链接伪类

链接伪类用于根据链接的不同状态（如未访问、悬停、点击等）为其添加样式，从而提升用户体验和界面交互性

使用场景：设置链接不同状态的样式

| 链接伪类    | 作用                                 |
| ----------- | ------------------------------------ |
| `a:link`    | 未访问链接的默认样式                 |
| `a:visited` | 已访问链接的样式                     |
| `a:hover`   | 鼠标悬停在链接上时的反馈             |
| `a:active`  | 链接被点击时的瞬时状态（按下到松开） |

调试工具查看伪类状态：强制显示状态

在浏览器开发者工具中，可通过右键元素 → `Force state`（强制状态）来预览不同伪类样式：

表格

| 操作步骤              | 说明                                                         |
| --------------------- | ------------------------------------------------------------ |
| 1. 右键目标元素       | 在开发者工具的 Elements 面板中，右键需要调试的 HTML 元素（如 `<a>` 标签） |
| 2. 选择 `Force state` | 在弹出的右键菜单中，找到并点击 `Force state` 选项            |
| 3. 选择伪类状态       | 展开子菜单后，可勾选需要强制显示的伪类，如：- `:active`：链接被点击时的瞬时状态- `:hover`：鼠标悬停时的状态- `:focus`：元素获得焦点时的状态- `:visited`：已访问链接的状态- `:focus-within`：元素或其子元素获得焦点时的状态- `:focus-visible`：键盘焦点可见时的状态 |



##### 用户行为伪类

用户以某种方式和文档交互的时候应用，这些用户行为伪类，有时候叫做动态伪类

使用场景：

- 鼠标经过元素的时候，修改样式
- 表单获得焦点的时候

| 动态伪类 | 作用                                 |
| -------- | ------------------------------------ |
| `:hover` | 鼠标经过元素时的样式                 |
| `:focus` | 获得焦点的元素（常见于表单输入元素） |

:hover 应用场景

- 不仅适用于链接 `<a>`，还可用于按钮、卡片、图片等任意元素，实现鼠标移入移出的交互反馈。
- 示例：`button:hover { background: #007bff; color: #fff; }`

:focus 应用场景

- 主要针对表单元素，如 `<input>`、`<textarea>`、`<select>` 等，当元素被选中（可输入 / 操作）时触发样式变化。
- 示例：`input:focus { border: 2px solid #28a745; outline: none; }`

使用注意

- 移动端适配：`:hover` 在触屏设备上可能无效果（无鼠标操作），需结合 `:active` 或触摸事件优化。
- 表单可访问性：`:focus` 样式需明显，方便键盘导航用户识别当前操作元素。

#### 结构伪类选择器

根据元素的位置选择

使用场景：选择第一个或者最后一个或者第n个元素

| 结构伪类        | 作用                                    |
| --------------- | --------------------------------------- |
| `:first-child`  | 匹配一组兄弟元素中的**第一个**元素      |
| `:last-child`   | 匹配一组兄弟元素中的**最后一个**元素    |
| `:nth-child(n)` | 匹配一组兄弟元素列表中的**第 n 个**元素 |

n的取值可以是

- 数字，从1开始
- 关键字。odd奇数 even偶数
- 公式

:nth-child(3n) ：3的倍数

：nth-child(n+2)  :第二个以及以后的元素

：nth-child(-n+3)  :前面三个元素

注意公式的n取值从0开始



#### 表单伪类选择器

针对表单元素的状态，比如表单禁用，选中复选框等

使用场景：

1. 表单按钮禁用的时候，调整颜色
2. 复选框选中修改样式

| 表单伪类    | 作用                                  |
| ----------- | ------------------------------------- |
| `:disabled` | 匹配处于**禁用状态**的表单元素        |
| `:checked`  | 匹配处于**选中状态**的单选 / 复选按钮 |

opacity：0-1 透明度



#### 伪元素选择器

选择元素的特定部分（使用双冒号：：）

使用场景

- 让表单里面的placeholder文字变颜色
- 做很多的修饰效果

选择特定部分：

| 伪元素选择器     | 作用                                            |
| ---------------- | ----------------------------------------------- |
| `::first-line`   | 选择块级元素的**首行文本**                      |
| `::first-letter` | 选择块级元素的**首字母 / 首字符**               |
| `::placeholder`  | 选择 `<input>` 或 `<textarea>` 的**占位符文本** |

**::first-line 注意事项**

- 仅作用于**块级元素**（如 `<p>`、`<div>`），行内元素（如 `<span>`）无效。

- 样式仅影响首行，换行后不会继承。

- 示例：

  ```
  p::first-line {
    color: #007bff;
    font-weight: bold;
  }
  ```

**::first-letter 注意事项**

- 仅作用于块级元素，首字母前不能有其他内容（如图片、符号）。

- 常用于首字下沉效果。

- 示例：

  ```
  p::first-letter {
    font-size: 2em;
    color: #dc3545;
    float: left;
  }
  ```

**::placeholder 注意事项**

- 用于修改输入框占位符的样式，不同浏览器可能需要前缀

- 示例：

  ```
  input::placeholder {
    color: #6c757d;
    font-style: italic;
  }
  ```





*插入内容：*

| 伪元素选择器 | 作用                               |
| ------------ | ---------------------------------- |
| `::before    | 元素内插入伪元素，作为第一个元素   |
| `::after`    | 元素内插入伪元素，作为最后一个元素 |

注意：

- `before` 和 `after` 是插入的**伪元素**，特性类似内联元素。
- `content` 属性是必须，不能省略，没有内容时空引号即可。
- 后期非常常用，比如小图标、各种装饰效果（后续会讲解）。



### 属性选择器

根据元素的属性特征来精准定位元素，从而实现更灵活的样式控制

使用场景：

1. 表单样式控制
2. 图标字体控制
3. 国际化语言适配等等

| 属性选择器   | 作用                                 |
| ------------ | ------------------------------------ |
| `[属性]`     | 匹配包含指定属性的元素               |
| `[属性=值]`  | 匹配属性值等于指定值的元素           |
| `[属性^=值]` | 匹配属性值以指定字符串**开头**的元素 |
| `[属性$=值]` | 匹配属性值以指定字符串**结尾**的元素 |
| `[属性*=值]` | 匹配属性值任意位置包含指定子串的元素 |
